## 1. Problem Statement

### 1.1 课程背景

学校要求完成"编译原理课程设计"，原始题目要求实现一个简单的 **PL/0 编译器**。课程同时提供一份改造课题：将 PL/0 改造成支持 **Rust 风格特性**（栈内存管理 + 借用检查），并对比 Rust 真实编译器的实现。

### 1.2 痛点

| 用户痛点 | 描述 |
|----------|------|
| 现有 PL/0 太弱 | 没有类型系统、没有 RAII、没有借用检查，不能体现"现代编译器"精髓 |
| 老师期望高 | 评分"良"/"优"的关键是体现对 Rust 编译原理的深入理解 |
| 学生不知从何下手 | Rust 编译器本身非常复杂（rustc ~30万行），需要找合适的参考和简化路径 |
| 完整实现不现实 | 4 周时间不可能从零写一个 Rust 子集编译器 |

### 1.3 我们的目标

在**保留原 PL/0 完全可用的前提下**，增量加入 Rust 核心特性：
- 类型系统（i8/i16/i32 + 借用）
- 借用检查（5 条规则）
- 栈内存管理（含悬垂借用检测）
- 通过 Qt GUI 可演示

让学生既"看得到"原 PL/0 的兼容性，又"看得见"Rust 风格特性的价值。

---

## 2. Solution

### 2.1 解决方案概览

**PL/0+ = PL/0 + Rust 风格特性的子集**

| 维度 | 方案 |
|------|------|
| 基础 | `PL0_compiler_with_LL1_and_QT`（与课程 PL0.exe 同源）|
| 策略 | **方案 A：增量式 Rust 化**（保留所有 PL/0 + 加 Rust 特性）|
| 范围 | 28 个测试用例，覆盖兼容性 + 借用规则 + 间接寻址 |
| 演示 | Qt GUI 加载测试用例 → 编译 → 报告 → 执行 |
| 时间 | 4 周，冲"优" |

### 2.2 核心用户场景

1. **老师验收** — 老师运行 PL0Plus.exe，演示 T1-T28
2. **学生演示** — 用 test1.pl0（PL/0 原生）证明兼容性
3. **报告展示** — 报告 6 章，词法/语法/借用/pcode 全部带原理图

### 2.3 价值主张

| 对谁 | 价值 |
|------|------|
| **老师** | 看到完整的"PL/0 → Rust 风格"渐进式改造，体现深度 |
| **学生（用户）** | 拿到"优" |
| **未来学生** | 这份 PRD + Spec 可作为"编译原理实验教材"参考 |

## 3. User Stories

### 3.1 兼容性用户故事

1. As a **PL/0 用户**，I want **PL0Plus 能运行所有原 PL/0 测试用例**（T1.PL0、E01.PL0 等），so that **保证原 PL/0 兼容性，零回归**。
2. As a **老师**，I want **PL0Plus 能用 Qt GUI 加载 PL/0 程序并执行**（参考 PL0.exe），so that **可以像原 PL/0 一样在 IDE 里调试**。

### 3.2 Rust 风格特性用户故事

3. As a **Rust 学习者**，I want **能写 `let x: i32 = 5;`** 的 Rust 风格声明，so that **可以对比 PL/0 的 `var` 和 Rust 的 `let`**。
4. As a **Rust 学习者**，I want **能写 `let mut x: i32 = 5;`** 的可变声明，so that **可以演示 Rust 的可变性控制**。
5. As a **Rust 学习者**，I want **能写 `&x` 创建不可变借用**，so that **可以演示 Rust 借用的基本概念**。
6. As a **Rust 学习者**，I want **能写 `&mut x` 创建可变借用**，so that **可以演示 Rust 的独占借用**。
7. As a **Rust 学习者**，I want **多个不可变借用可以同时存在**（`let r1 = &x; let r2 = &x;`），so that **可以演示 Rust 借用规则 1**。
8. As a **Rust 学习者**，I want **同一个变量不能同时有可变和不可变借用**（`&x` 和 `&mut x` 冲突），so that **可以演示 Rust 借用规则 3**。
9. As a **Rust 学习者**，I want **同一个变量不能同时有两个可变借用**（两个 `&mut x`），so that **可以演示 Rust 借用规则 2**。
10. As a **Rust 学习者**，I want **能写 `*r` 解引用读取借用值**，so that **可以演示通过借用访问原值**。
11. As a **Rust 学习者**，I want **能写 `*m := 10` 通过可变借用修改原值**，so that **可以演示 `&mut` 的最大用途**。
12. As a **Rust 学习者**，I want **借用者出作用域时借用自动释放**（begin/end RAII），so that **可以演示 Rust 的 NLL（非词法生命周期）**。
13. As a **Rust 学习者**，I want **检测悬垂借用**（内层变量被外层借用），so that **可以演示 Rust 借用检查器防悬垂指针**。

### 3.3 错误处理用户故事

14. As a **PL/0+ 用户**，I want **借用错误时能给出清晰错误信息**（错误码 + 行号 + 修复建议），so that **可以快速定位问题**。
15. As a **PL/0+ 用户**，I want **错误恢复后能继续编译后面的代码**（恐慌模式），so that **一次能看到所有错误，不是改一个错再看下一个**。

### 3.4 报告用户故事

16. As a **老师**，I want **报告中包含完整的 EBNF 文法**，so that **能看到 Rust if 条件语句的完整语法定义**。
17. As a **老师**，I want **报告中包含 mrustc 源码分析**，so that **能证明参考了真实 Rust 编译器实现**。
18. As a **老师**，I want **报告中包含 pcode 指令集扩展说明**，so that **能理解 LEA/LODI/STOI/REL 4 条新指令的设计**。
19. As a **老师**，I want **报告中包含借用检查算法详解**，so that **能评估算法的正确性和完整性**。
20. As a **老师**，I want **报告中包含栈帧布局图**，so that **能直观看到借用与栈帧的交互**。

### 3.5 演示用户故事

21. As a **演示者**，I want **PL0Plus.exe 启动后能直接看到代码编辑器**，so that **可以快速加载测试用例**。
22. As a **演示者**，I want **点击"运行"后能显示编译结果和输出**，so that **可以演示 28 个测试用例的运行效果**。
23. As a **演示者**，I want **看到 Token 列表、语法分析过程、pcode**，so that **能展示编译器各阶段工作**。

## 4. Implementation Decisions

### 4.1 决策摘要

| # | 决策 | 理由 |
|---|------|------|
| 1 | **方案 A：PL/0 增量式 Rust 化** | 改动可控、兼容原测试、评分有保障 |
| 2 | 基础 = `PL0_compiler_with_LL1_and_QT` | 与课程 PL0.exe 同源，便于老师验收 |
| 3 | 一份合并报告（内容 2 + 内容 3） | 结构紧凑 |
| 4 | 1 个月时间，冲"优" | 充裕 |
| 5 | Qt GUI 演示 | 与课程 PL0.exe 一致 |
| 6 | 完全保留 PL/0 原功能 | 不破坏兼容性 |
| 7 | 静态栈分配（方案 A） | 简单、pcode 简洁 |
| 8 | 报告中文 | 与课程要求一致 |
| 9 | 28 个测试用例 | 充分覆盖 |
| 10 | 词法层加 8 个 Token | let/mut/i8/i16/i32/&/&mut/: |
| 11 | `&mut` 单 token | 词法简单，maximal-munch |
| 12 | 类型强制标注，无推断 | 教学友好 |
| 13 | C++11 `enum class` | 强类型 |
| 14 | 借用双轨制（snapshot per-statement + release per-scope）| 错误粒度 + Rust 语义 |
| 15 | `begin/end` 创建块作用域 | 让借用生命周期可控 |
| 16 | procedure 隔离借用 | 防悬垂 |
| 17 | 纯 RAII 自动释放 | 不引入 `drop` 关键字 |
| 18 | 借用禁止跨 procedure | PL/0 procedure 无参数和返回值 |
| 19 | 错误码 E0001-E0099 | 统一规范 |
| 20 | 错误恢复用恐慌模式 | 一次报告多个错误 |
| 21 | 错误上限 50 | 防死循环 |
| 22 | 新增 4 条 pcode 指令 | LEA/LODI/STOI/REL |
| 23 | REL 0 N 独立指令 | OPR A 域已被占用 |
| 24 | 静态借用检查 + 运行时越界检查 | 编译时 + 运行时双重 |
| 25 | 类型映射 integer→I32 | 类型系统一致 |

### 4.2 模块清单

| 模块 | 作用 | 状态 |
|------|------|------|
| **compiler/lexer** | 词法分析（35+8 Token）| 扩展 |
| **compiler/parser** | 语法分析（LL(1)）| 扩展 |
| **compiler/grammar** | 文法 + First/Follow | 扩展 |
| **compiler/symbolTable** | 符号表（含 DeclKind）| 扩展 |
| **compiler/borrow_checker** ⭐ | 借用检查（5 条规则）| 新增 |
| **compiler/ast** | 抽象语法树 | 扩展（+lvalue、+deref）|
| **compiler/token** | Token 数据 | 扩展（+8 个）|
| **compiler/error** | 错误处理 | 扩展 |
| **vm/pl0vm** | 虚拟机 | 扩展（+4 指令）|
| **vm/instruction** | 指令定义 | 扩展 |
| **vm/stack** | 栈管理 | 扩展 |
| **ui/mainwindow** | Qt GUI | 保留 |
| **ui/codeeditor** | 代码编辑器 | 保留 |
| **tests/** | 28 个 .pl0 测试 | 新增 |
| **docs/** | 报告 + 设计决策 | 新增 |

### 4.3 不在范围内（Out of Scope）

- ❌ 多级解引用 `**p`（文法层堵死）
- ❌ 类型推断（强制标注）
- ❌ `f32/f64` 浮点
- ❌ 显式 `drop` 关键字
- ❌ lifetime 标注
- ❌ 闭包、trait、泛型
- ❌ 多线程借用检查
- ❌ 自定义分配器
- ❌ 编译器自举

## 6. Out of Scope

明确**不做**的功能（避免范围蔓延）：

1. **多级解引用** `**p` — 文法层堵死
2. **类型推断** — 强制标注
3. **f32/f64 浮点** — 只做整数
4. **显式 drop 关键字** — 纯 RAII
5. **lifetime 标注** — 范围太复杂
6. **trait、泛型、闭包** — 远超课程范围
7. **多线程借用检查** — 单线程即可
8. **优化**（如内联、寄存器分配）— 教学编译器不需要
9. **与 rustc 源码完全对应** — 只参考 mrustc 部分模块
10. **跨平台支持** — Windows 优先

## 8. Risks & Mitigations

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| 借用检查边界条件漏检 | 中 | 高 | 充分测试 + 错误恢复 |

| 报告质量不过关 | 中 | 高 | 边写代码边记要点 |
| 时间不够 | 低 | 高 | 砍掉非核心测试用例 |
